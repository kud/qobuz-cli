`qobuz-cli` doesn't stream audio itself вЂ” the official Qobuz app stays the player. Two mechanisms drive it: **deep links** (`open` / `url`) that point the app at an item, and **media keys** (`play` / `next` / `previous`) that control whatever is currently playing.

## рџ”— Deep links вЂ” `open` and `url`

Both take a `<type>` (`album`, `track`, `playlist`, or `artist`) and an `<id>`, and build an `open.qobuz.com` link.

### `qobuz url <type> <id>` (alias `copy-url`)

Copies the deep link to the **clipboard** (via `pbcopy`) instead of opening it вЂ” handy for pasting into a chat or note. It prints a confirmation showing exactly what was copied:

```console
$ qobuz url album 0634904032432
вњ“ copied https://open.qobuz.com/album/0634904032432
```

`copy-url` is an alias for the same command. On a system without `pbcopy` (i.e. non-macOS) it falls back to printing the bare link to stdout, so it still composes in a pipe.

### `qobuz open <type> <id>`

Opens the item in the Qobuz app (falls back to the web player if the app isn't installed).

```console
$ qobuz open album 0634904032432
opening https://open.qobuz.com/album/0634904032432
```

An unrecognised `<type>` exits non-zero with `type must be one of: album, track, playlist, artist`.

## вЏЇпёЏ Media keys вЂ” `play`, `next`, `previous`, `forward`, `rewind`

These post synthetic media-key events to the system, exactly like the keys on a keyboard вЂ” so they control the **active** player (Qobuz, when it's playing). There's no "now playing" readout: macOS locks that down, and Qobuz exposes no playback API.

```console
$ qobuz play       # toggle play / pause
$ qobuz next       # skip to the next track
$ qobuz previous   # go to the previous track  (alias: prev)
$ qobuz forward    # fast-forward within the current track  (alias: ff)
$ qobuz rewind     # rewind within the current track  (alias: rew)
```

**Skip vs scrub.** `next` / `previous` jump between whole tracks; `forward` / `rewind` scrub _within_ the current track. They're distinct media keys (`NX_KEYTYPE_FAST` / `NX_KEYTYPE_REWIND`), so whether they seek depends on the active player honouring them.

### рџЌЏ macOS only + Accessibility permission

Playback control is **macOS-only** and posts HID events through a tiny Swift helper bundled with the package (compiled on first use via `swiftc`). For the events to take effect, your **terminal app must be granted Accessibility permission**:

> **System Settings в†’ Privacy & Security в†’ Accessibility** в†’ enable your terminal (Terminal, iTerm, Ghostty, вЂ¦).

The spawned helper inherits the terminal's permission. Without it, the command exits cleanly but **nothing happens** вЂ” if a key seems to do nothing, this is almost always why. (Compilation needs the Xcode command-line tools: `xcode-select --install`.)
